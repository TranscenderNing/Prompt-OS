## PromptOS Docker 化（v0.1）
- `backend/Dockerfile`：python:3.12-slim（避开 3.14 的 pydantic wheel 问题），`PROMPTOS_DB=/data/promptos.db`
- `frontend/Dockerfile`：多阶段，node:20-alpine 构建 + nginx:1.27-alpine 托管，nginx 反代 /api → backend:8000
- `docker-compose.yml`（生产）：backend 健康检查 + frontend depends_on service_healthy + 命名卷 `promptos-data` 持久化 + 外部端口由 `PROMPTOS_PORT` 控制（默认 8080）
- `docker-compose.dev.yml`（开发）：bind mount 源码，uvicorn --reload + vite dev；Vite 代理目标由 `VITE_API_BASE` 环境变量切换
- `.env.example`：PROMPTOS_PORT / PROMPTOS_LLM_*；前端浏览器 localStorage Key 优先，fallback 到后端 env

## PromptOS CI/CD（v0.1）
- `.github/workflows/ci.yml`：4 个 job（backend-test → frontend-build → docker buildx → release）
  - backend-test：起 uvicorn 后 curl 冒烟 health/list/render/commit/ai
  - frontend-build：tsc + vite build，上传 dist artifact
  - docker：QEMU + Buildx 多架构（amd64+arm64），metadata-action 按分支/PR/tag 自动打 tag，GHA 缓存加速，推 GHCR
  - release：tag `v*.*.*` 触发，打包 `docker-compose.release.yml + .env.example + README` 成 tgz 附到 GitHub Release
- `docker-compose.release.yml`：纯 image 拉取版，占位符 `ghcr.io/OWNER/REPO-{backend,frontend}:${IMAGE_TAG:-latest}`
- `.github/dependabot.yml`：每周扫 actions/pip/npm/docker 更新
